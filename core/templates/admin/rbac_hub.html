🔗 Intégration

PostgREST utilise les rôles PostgreSQL pour enforcer les permissions:

  • Les tokens JWT incluent le rôle de l'utilisateur
  • PostgREST exécute les requêtes avec les permissions du rôle
  • Accès refusé (403) si le rôle n'a pas les droits

📋 Format de la matrice

La matrice RBAC doit être en CSV ou Excel avec:

  • Colonne 1: Nom de la table
  • Colonnes suivantes: Codes de permission (C/R/U/D/V/-)
  • Codes: C=Create, R=Read, U=Update, D=Delete, V=Validate
